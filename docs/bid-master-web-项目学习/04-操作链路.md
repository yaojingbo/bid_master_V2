# 维度4：用户操作链路

## 操作：登录工作台

1. 用户在 LoginForm（路径：app/(auth)/login/page.tsx）中输入邮箱密码，点击"登录"按钮
2. 前端 useAuthStore().login()（路径：src/frontend/stores/auth-store.ts）调用 authLogin（路径：src/frontend/lib/auth-api.ts）发出 POST /api/auth/login，请求体包含 email、password
3. 请求进入 src/backend/app/api/auth.py 的 /login handler，做 Pydantic 字段校验、限流检查
4. 调用 pg_storage.get_user_by_email（路径：src/backend/app/infrastructure/pg_storage.py）查询 users 表，bcrypt 校验密码
5. 写数据库表 audit_logs（路径：drizzle/0000_icy_maestro.sql），记录 action=login；调用 create_access_token / create_refresh_token（路径：src/backend/app/utils/jwt.py）颁发 JWT
6. 返回 { access_token, user }，accessToken 存内存，refreshToken 通过 Next Route 写 httpOnly cookie
7. 前端在 WorkbenchLayout（路径：src/frontend/components/layout/WorkbenchLayout.tsx）里跳转到 /workbench，Sidebar 渲染用户信息

## 操作：在 /extract 上传并提取要素

1. 用户在 FileUploader（路径：src/frontend/components/file-upload/FileUploader.tsx）中拖拽 PDF，点击"开始提取"
2. 前端 useFileUpload().upload(file)（路径：src/frontend/hooks/useFileUpload.ts）调用 uploadFile（路径：src/frontend/lib/api.ts）发出 POST /api/files/upload（multipart）
3. 请求进入 src/backend/app/api/files.py 的 /upload handler，FileService.validate_file 检查扩展名/大小
4. 调用 EncryptionService.encrypt（路径：src/backend/app/services/encryption_service.py，Fernet）+ StorageService 写盘
5. 写数据库表 tender_documents（路径：drizzle/0000_icy_maestro.sql），status=uploading → ready，返回 { id, filename, size }
6. 前端 handleExtract（路径：app/(main)/extract/page.tsx）调用 runExtractTask（路径：src/frontend/lib/extract-runner.ts）发出 POST /api/extract/element SSE，body 包含 file_id、template_id、elements[]
7. 请求进入 src/backend/app/api/extract.py 的 /element handler，返回 StreamingResponse
8. 后端调用 extract_service（路径：src/backend/app/services/extract_service.py）：解密 PDF → extract_text_from_content 抽文本 → 文本过少走 ocr_service → PromptBuilder 构造 prompt（路径：src/backend/app/services/prompt_builder.py）→ LiteLLMService 流式调 LLM → 解析 JSON → 流式 SSE 推送 progress/content/done
9. 写数据库表 analysis_results（type=element_extract、status、result jsonb）+ extracted_elements（每条要素一行，含 confidence/position）
10. 前端 extract-runner.ts 解析 SSE 事件，写入 task-store；TaskProgress（路径：src/frontend/components/ui/TaskProgress.tsx）显示 phase + 百分比，MarkdownPreview（路径：src/frontend/components/ui/MarkdownPreview.tsx）流式渲染提取结果

## 操作：在 /simulate 创建并跑完 4 步

1. 用户在 SimulateFlow（路径：src/frontend/components/simulate/SimulateFlow.tsx）中选文件，填写 project_name/project_type/region，点击"创建任务"
2. 前端 create（路径：app/(main)/simulate/page.tsx）发出 POST /api/simulate/create，body 包含 file_id、project_name、project_type、region
3. 请求进入 src/backend/app/api/simulate.py 的 /create handler，pg_storage.add_simulate 写 simulate_tasks 表，返回 { task_id }
4. 前端逐步发出 POST /api/simulate/{taskId}/step/1..4 SSE
5. step 1（PDF 转换）：FileService 读 + decrypt → PDF 转换（pdf2docx）→ 写中间产物
6. step 2（要素提取）：复用 extract_service（同上）→ 写 analysis_results + extracted_elements
7. step 3（对比分析）：PromptBuilder 构造对比 prompt → LiteLLMService 流式 LLM → 写 analysis_results（type=compare）
8. step 4（模拟编制）：构造编制 prompt（含 project 参数）→ LiteLLMService 流式生成投标文件草稿 → 写 simulate_tasks.result
9. 每步 SSE 事件流，前端 streamStep()（路径：app/(main)/simulate/page.tsx）解析并写入 task-store
10. TaskProgress 显示 4 步进度，MarkdownPreview 流式渲染最终投标草稿

## 操作：在 /statistics 上传 Excel 并查看 AI 分析

1. 用户在 ExcelUploader（路径：src/frontend/components/statistics/ExcelUploader.tsx）中拖拽开标 Excel
2. 前端 useFileUpload().upload(file) 调 uploadFile 发出 POST /api/files/upload（multipart）
3. 请求进入 src/backend/app/api/files.py 的 /upload handler（见登录链路 3-5 步），写 tender_documents，返回 { id }
4. 前端发出 POST /api/statistics/analyze/upload，body 包含 file_id
5. 请求进入 src/backend/app/api/statistics.py 的 /analyze/upload handler（约第 625 行），openpyxl 解析 Excel
6. 调用 StatisticsService.calculate_price_statistics（路径：src/backend/app/services/statistics_service.py）— 纯计算：排名、CV、降价幅度
7. 写数据库表 analysis_results（type=opening_analysis）+ statistics（data jsonb）
8. 前端收到 { analysis_id } 后立即发出 POST /api/statistics/analyze/comprehensive/upload/start，body 包含 analysis_id
9. 请求进入 /analyze/comprehensive/upload/start handler，创建后台任务，立即返回 { task_id }
10. 后台线程：PromptBuilder 构造综合分析 prompt（路径：src/backend/app/services/prompt_builder.py）→ LiteLLMService 流式调 LLM → 进度写 task_manager → 完成后写 analysis_results.result.ai_analysis
11. 前端轮询 GET /api/statistics/analysis-task/{task_id} 拿到 ai_analysis 文本
12. StatisticsCards（路径：src/frontend/components/statistics/StatisticsCards.tsx）显示数字指标，PriceRankingsTable 渲染排名，MarkdownPreview 渲染 AI 综合分析

## 操作：在 /database 删除一条文件记录

1. 用户在 DatabasePanel（路径：src/frontend/components/database/DatabasePanel.tsx）的 files Tab 找到目标行，点击行尾的删除按钮
2. 前端 FileActions（路径：src/frontend/components/database/FileActions.tsx）调 deleteFile（路径：src/frontend/lib/data-api.ts）发出 DELETE /api/data/files/{id}
3. 请求进入 src/backend/app/api/database.py 的 /data/files/{id} DELETE handler，校验用户权限
4. 调用 pg_storage.delete_document 删除 tender_documents 行（ON DELETE cascade 同时清理 analysis_results、extracted_elements、statistics、download_logs）
5. 同时调用 StorageService.delete 删除加密文件
6. 返回 { success: true }
7. 前端在 DatabasePanel 中重新调 listFiles（GET /api/data/files）刷新列表，删除行消失

链路结构相同操作（不再展开）：

- /database 下载文件：调 GET /api/data/files/{id}/download → StorageService decrypt → FileResponse
- /database 预览文件：调 GET /api/data/files/{id}/preview → 解析前 5KB 文本 → JSON
- /database 批量下载：调 POST /api/data/files/batch-download → 多文件打包 zip（jszip）
- /settings 保存 API Key：调 POST /api/api-keys → EncryptionService.encrypt → 写 ai_configurations
- /settings 测试连接：调 POST /api/settings/test → LLMService.test_connection 发最小请求
- /logs 清空日志：调 DELETE /api/logs → 清 log_entries
- CLI 设备授权：CLI 调 /cli-auth/start → 浏览器 /cli-auth 授权 → CLI 调 /cli-auth/poll 拿 token
