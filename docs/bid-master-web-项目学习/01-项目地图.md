项目名称: bid-master-web

## 技术栈

TypeScript（前端）+ Python（后端）+ PostgreSQL（数据库）

- 前端：Next.js 15（App Router）+ React 19 + Tailwind 4 + Radix UI + Zustand + TanStack Query
- 后端：FastAPI（Python 3.12）+ LiteLLM（多 AI 供应商）+ cryptography（文件加密）
- 数据库：PostgreSQL + Drizzle ORM（TypeScript schema）
- 测试：Vitest（单元/集成）+ Playwright（E2E）
- 部署：Vercel（前端）+ Railway（后端）

## 入口文件

src/backend/app/main.py — FastAPI 应用入口，注册 11 个路由、middleware、CORS、lifespan 自动建表

src/backend/app/cli.py — `bidmaster` 命令行入口（`npm run cli`），支持本地与远程两种模式

app/layout.tsx — Next.js 根布局（Tailwind、字体、Sidebar 全局壳）

app/(main)/page.tsx — 营销首页（4 个特性卡片 + 安装命令）

src/db/schema.ts 或 drizzle/0000_icy_maestro.sql — 数据库 schema 入口（drizzle-kit 自动生成 SQL）

## 目录结构

app/ — Next.js 15 App Router 页面（(auth) 登录流 + (main) 工作台流 + api Next 路由代理）

src/frontend/ — 前端共享模块（components/hooks/stores/lib/types）

src/backend/ — FastAPI 后端（app/api 路由、app/services 业务、app/models schema、app/infrastructure 基础设施）

src/db/ — 数据库 Schema 与 TypeScript 类型

drizzle/ — Drizzle ORM 生成的 SQL 迁移文件

middleware.ts — Next.js 中间件：未登录访问保护页跳 `/login`

tests/ — Python 单元/集成测试（pytest）+ TypeScript 测试（vitest）

e2e/ — Playwright 端到端测试

docs/ — 文档中心（research/bug-fix-summary/plan/error-log/guide）

spec/{ai,hi}/ — AI 规约草稿与人工确认规约

.42cog/ — 认知敏捷法元数据（meta/real/cog）

.42plugin/ — 本地技能库；.42plugin.yml — 插件清单

## 关键配置

.env.local — 前端环境变量（API 地址、AI 供应商 Key 等敏感配置），不进入 Git

.env — 后端环境变量（数据库连接、JWT 密钥、LLM 供应商 Key）

next.config.js — Next.js 配置（rewrites：把 `/api/*` 代理到 FastAPI）

vercel.json / backend.railway.json — 部署配置

Makefile — 启动/重启前后端服务的快捷命令（`make dev` / `make restart`）

tailwind.config.ts / postcss.config.mjs — Tailwind 4 配置

tsconfig.json / next-env.d.ts — TypeScript 配置

drizzle.config.ts — Drizzle ORM 配置（数据库连接、迁移目录）
