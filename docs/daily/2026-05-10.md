# 日常开发记录

## 2026-05-10

### 1. 单元测试修复

**问题**：Vitest 运行前端测试时，`@/lib/...` 路径无法解析。

**原因**：
- `vitest.config.ts` 中 `@` 别名指向 `src/frontend`
- 但测试文件在 `src/frontend/__tests__/`
- 需要导入的 `src/lib/validations/...` 路径不正确

**修复**：
1. 修正测试文件的 import 路径：
   - `src/frontend/__tests__/lib/utils.test.ts`: `../../lib/utils` → `../../lib/utils`
   - `src/frontend/__tests__/components/file-upload/FileUploader.test.tsx`: `../../components/file-upload/FileUploader` → `../../../components/file-upload/FileUploader`
2. 创建 `src/lib/utils.ts` 重新导出 `cn` 函数
3. 在 `src/lib/validations/ai-config.ts` 和 `analysis.ts` 补充缺失的 Zod schemas
4. 修正 `ElementCard` 组件的 `confidence` 显示（0.95 → 95%）

**测试结果**：66 tests passed ✓

### 2. Next.js 配置修复

**问题**：Next.js 找不到 `app` 目录。

**原因**：
- 新的 Next.js 15 项目结构，`app` 目录在 `src/frontend/` 下
- 但 Next.js 默认在项目根目录查找

**修复**：
1. 将 `src/frontend/app/` 复制到项目根目录 `/app/`
2. 简化 `next.config.js`，移除无效的 `experimental.srcDir` 配置

### 5. API 路径问题

**问题**：前端 `useFileUpload.ts` 等文件导入 `@/lib/api` 失败。

**原因**：`@/lib/api` 指向 `src/lib/api.ts`，但该文件 re-export 路径 `../../frontend/lib/api` 不正确。

**修复**：
1. 在 `src/frontend/lib/api.ts` 创建完整的 API 客户端（包含 `uploadFile`, `listFiles`, `deleteFile`, `downloadFile`, `extractElements`, `downloadBlob`）
2. 修正 `src/lib/api.ts` 的 re-export 路径为 `../frontend/lib/api`
3. 修正 `src/lib/utils.ts` 的 re-export，添加 `formatDate`, `formatFileSize`, `downloadBlob`
