## 1. 产品环境

**名称：** Bid Master Web

**标语：** 招投标全流程智能分析环境

**版本：** 0.1.0

**环境描述：**

本产品为招投标从业人员创建一个**动作可能性空间**，支持招标文件智能要素提取、模拟编制和开标报价分析三大核心场景。环境向智能体（人类和 AI）提供明确的感知-动作耦合，实现招投标工作流的效率提升。

**主要智能体：**

| 智能体 | 动作能力 |
|--------|----------|
| 投标方 | 上传招标文件、触发分析、查看结果、下载报告 |
| 招标方/代理机构 | 管理分析任务、导出数据 |
| 评标专家 | 审阅分析结果 |
| AI 智能体 | 调用 API、执行批量分析、生成结构化报告 |

**核心 Affordance：**

| 层级 | Affordance | 含义 |
|------|------------|------|
| 初级 | 文件上传 | 拖拽或选择招标文件（PDF/Markdown） |
| 初级 | 分析触发 | 一键启动要素提取/开标分析/模拟编制 |
| 次级 | 结果对比 | 多文件分析结果并排对比 |
| 次级 | 报告导出 | 生成结构化分析报告 |
| 潜在 | 批量处理 | 多个文件批量分析 |
| 潜在 | 团队协作 | 多人共享分析项目和结果 |

## 2. Affordance 目录

### 2.1 初级 Affordance（立即可感知）

#### AFF-01: 文件上传

**层级：** 初级
**启用动作：** 将本地招标文件上传至服务端

| 环境属性 | 值 | 可被感知方式 |
|----------|-----|-------------|
| 视觉提示 | 拖拽区域 + 文件选择按钮 | 人类 |
| 语义标记 | `<input type="file" accept=".pdf,.md">` | AI 智能体 |
| 交互模式 | 拖拽 或 点击选择 | 两者 |

**感知-动作耦合：**

| 智能体 | 感知 | 动作 | 反馈 |
|--------|------|------|------|
| 人类 | 看到拖拽区域和提示文字 | 拖拽文件或点击选择 | 文件名显示 + 上传进度条 |
| AI | 检测到 file input 元素 | 调用文件上传 API | 上传结果响应 |

**约束：**
- 仅允许 PDF、Markdown、doc、docx、excel 文件（服务端验证）
- 文件大小限制：50MB（前端提示 + 服务端校验）
- 上传后文件加密存储（来自 real.md）

---

#### AFF-02: 要素提取

**层级：** 初级
**启用动作：** 从招标文件中提取五大要素（资质要求、评标办法、业绩门槛、定标方法、合同条款）

| 环境属性 | 值 | 可被感知方式 |
|----------|-----|-------------|
| 视觉提示 | 分析按钮 + 加载动画 | 人类 |
| 语义标记 | `data-action="extract-elements"` | AI 智能体 |
| 交互模式 | 点击按钮触发，SSE 流式输出 | 两者 |

**感知-动作耦合：**

| 智能体 | 感知 | 动作 | 反馈 |
|--------|------|------|------|
| 人类 | 看到"提取要素"按钮 | 点击触发 | 实时流式显示提取结果 |
| AI | 检测到 action 标记 | POST /api/extract | SSE 流式响应 |

**约束：**
- 统计分析在服务端完成（来自 real.md）
- API 密钥通过环境变量配置（来自 real.md）

**故事主题：** 将散落的招标信息转化为结构化的要素清单

---

#### AFF-03: AI 配置

**层级：** 初级
**启用动作：** 配置和切换 LLM 供应商

| 环境属性 | 值 | 可被感知方式 |
|----------|-----|-------------|
| 视觉提示 | 供应商下拉选择 + 连接测试按钮 | 人类 |
| 语义标记 | `data-config="ai-provider"` | AI 智能体 |
| 交互模式 | 选择供应商 + 输入 API Key | 两者 |

**感知-动作耦合：**

| 智能体 | 感知 | 动作 | 反馈 |
|--------|------|------|------|
| 人类 | 看到供应商列表和测试按钮 | 选择并测试连接 | 显示连接成功/失败状态 |
| AI | 检测到 config 标记 | 调用测试 API | 返回测试结果 |

**约束：**
- API Key 不存储在前端，仅保存在环境变量（来自 real.md）

## 3. 环境约束

### 3.1 物理约束

| 约束 | 对 Affordance 的影响 |
|------|---------------------|
| 浏览器兼容性 | 支持现代浏览器（Chrome、Firefox、Safari、Edge 最近 2 年版本） |
| 网络延迟 | 文件上传和分析接口需处理网络异常 |
| 移动端支持 | 响应式布局，关键操作可在移动端完成 |

### 3.2 结构约束

| 约束 | 对 Affordance 的影响 |
|------|---------------------|
| 数据模型 | document-analysis 一对多关系 |
| 用户角色 | 投标方、招标方、评标专家权限区分 |
| 文件格式 | 仅支持 PDF、Markdown |

### 3.3 安全约束（来自 real.md）

| 约束 | Affordance 限制 |
|------|----------------|
| API 密钥管理 | AI 配置仅支持环境变量，不存储在数据库 |
| 文件加密 | 上传文件必须加密存储 |
| 统计计算 | 报价统计必须在服务端完成 |

## 4. 感知通道

### 4.1 视觉 Affordance（人类）

| 元素 | 可感知动作 | 设计模式 |
|------|-----------|----------|
| 上传区域 | 可拖拽上传 | 虚线边框 + 图标 + 提示文字 |

### 4.2 语义 Affordance（AI）

| 元素 | 语义标记 | 可感知动作 |
|------|---------|-----------|
| 文件上传 | `data-action="upload"` | 文件上传 API |
| 分析触发 | `data-action="analyze"` | 分析 API |
| 结果获取 | `data-action="result"` | 获取结果 API |
| 导出功能 | `data-action="export"` | 导出 API |

### 4.3 跨模态 Affordance

| Affordance | 人类通道 | AI 通道 | 统一动作 |
|------------|---------|--------|---------|
| 上传文件 | 拖拽/点击 | File API | 文件存储 |
| 触发分析 | 点击按钮 | POST API | 启动分析 |
| 获取结果 | 观察显示 | GET API | 数据获取 |
| 导出报告 | 点击下载 | 下载文件流 | 文件导出 |

## 7. 非 Affordance（明确不支持的动作）

| 不支持的动作 | 原因 |
|-------------|------|
| 直接编辑上传的原始文件 | 安全性考虑，防止篡改 |
| 招标方直接修改分析结果 | 保证分析客观性 |
| 多人同时编辑同一项目 | MVP 阶段不支持协作功能 |
| 移动端完整功能 | 复杂分析操作仅限桌面端 |

## 9. 附录

### 9.1 Affordance 理论参考

基于 Gibson 的 affordance 理论：环境向智能体提供动作可能性。产品在 AI 时代既是人类的运行环境，也是 AI 智能体的运行环境。

### 9.2 术语表

| 术语 | 定义 |
|------|------|
| Affordance | 环境属性与智能体动作能力的耦合关系 |
| SSE | Server-Sent Events，服务端向浏览器单向推送 |
| MAS | Minimum Affordance Story，最小动作可能性故事 |
| LiteLLM | 统一封装多个 LLM 供应商的 Python 库 |

### 9.3 里程碑映射

| 阶段 | 交付的 Affordance |
|------|------------------|
| MVP | AFF-01（文件上传）、AFF-02（要素提取）、AFF-03（AI 配置） |
| v1.0 | AFF-04（结果对比）、AFF-07（模拟编制）、AFF-08（开标分析） |
| v2.0 | AFF-05（报告导出）、AFF-06（批量处理）、AFF-09（团队协作） |
