【问题1】批量要素提取"数值过长"
根因：extracts.file_id 为 VARCHAR(64)，批量模式下传入 ",".join(file_ids) 拼接多文件ID，超出长度限制
修复：
  - db_schema.py: extracts.file_id VARCHAR(64) → TEXT；添加 status VARCHAR(30) 列；添加迁移4-7
  - pg_storage.py: add_extract INSERT 加入 status 列

【问题2】模拟编制进行到对比分析时任务中止
根因：simulate_service.py 的 run_step2/3/4_stream 中，LLM 报错且 result_holder["text"] 为空时，finally 块缺少 else 分支，任务状态永远卡在当前步骤
修复：
  - simulate_service.py: 三个 step stream 的 finally 块补全 else 分支，LLM 无输出时更新 status="error"
  - lite_llm.py: httpx 超时 60s → 180s，减少复杂分析任务超时失败

【问题3】开标分析综合AI分析任务停滞
根因：openings 表缺少 ai_analysis TEXT 和 status VARCHAR(20) 列；add_opening INSERT 不包含这两个字段；update_opening 动态 SQL 引用不存在的 ai_analysis 列导致 PostgreSQL 错误静默失败
修复：
  - db_schema.py: openings 表添加 ai_analysis TEXT、status VARCHAR(20) 列；迁移7
  - pg_storage.py: add_opening INSERT 加入 ai_analysis 和 status 列

【问题4】门槛分析进度条完成但无分析结果
根因：门槛分析的 LLM 返回叙述性文本而非 {"elements":[...]} JSON。后端 _stream_llm_with_progress 在 JSON 解析无 element 时只发 done 事件不发 element 事件，前端只在 extractedElements 非空时显示结果
修复：
  - extract_service.py: JSON 解析无 element、文本格式、JSON 解析失败三条路径，均改为将完整文本作为 {"name":"分析结果","content":文本} 单个 element 发出
  - extract/page.tsx: 结果展示条件放宽——extractedElements 为空但 isDone 且 streamRawText 有内容时也显示分析结果
