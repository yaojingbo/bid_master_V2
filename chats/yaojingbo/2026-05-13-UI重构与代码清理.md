# 2026-05-13 UI 重构与代码清理

## 一、发现并解决项目双代码源问题

### 问题
项目存在两套 `app/` 目录：根目录 `app/`（Next.js 实际运行）和 `src/frontend/app/`（规约文档指定但无效）。之前所有改动都在 `src/frontend/app/` 中，从未生效。

### 解决
- 删除 `src/frontend/app/`（冗余页面代码）
- 删除 `src/lib/`（薄封装层，合并到 `src/frontend/lib/`）
- 保留根 `app/` 作为唯一页面目录
- 更新 `tsconfig.json`：`@/lib/*` → `src/frontend/lib/*`
- 更新 `code.spec.md`：页面路径从 `src/frontend/app/` → `app/`
- 更新 `CLAUDE.md` 项目结构描述

## 五、布局系统

### 内容区留白
经过多轮调整最终方案：
- `max-w-3xl` (768px) + `flex justify-center` 居中
- `py-16` (64px) 垂直留白
- `px-8` (32px) 水平内边距
- 目标：内容"漂浮"在大量空白中，不紧凑

### 全局 CSS Token
```css
--color-background: #fafafa;
--color-foreground: #1a1a1a;
--color-muted: #f5f5f5;
--color-muted-foreground: #999999;
--color-border: #e8e8e8;
--color-card: #ffffff;
```

所有组件统一使用语义化 token（`text-foreground`、`border-border` 等），消除硬编码色值。
